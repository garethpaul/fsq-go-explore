# FSQ User Cache Key Boundary

status: completed

## Context

Authenticated handlers read the `fsq` cookie and use it as the memcache key for
the cached Foursquare user. Valid keys are generated by `fsq.GetUserKey` as a
`user:` namespace plus a SHA-256 hex digest, so malformed cookie values should
be rejected before App Engine context or memcache work starts.

## Objectives

- Validate user cache keys before memcache lookup.
- Accept only generated `user:` SHA-256 digest keys.
- Reject blank, wrong-namespace, non-hex, and oversized cookie values.
- Add focused tests for the cache-key boundary.
- Extend the static baseline and docs so the cookie/memcache boundary remains
  visible.
